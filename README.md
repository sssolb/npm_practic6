\## Звіт про технічну проблему (EPERM Error)

Під час тестування засобами `eu-node-basics verify` на ОС Windows виникла системна помилка:

`FAIL EPERM, Permission denied: \\\\?\\C:\\Users\\...\\AppData\\Local\\Temp\\...`



\*\*Скріншот помилки:\*\*

!\[FailEperm](FailEperm.png)##



\*\*Технічна причина:\*\*

Помилка пов'язана з механізмом \*\*File Locking\*\* у файловій системі NTFS (Windows). Утиліта для верифікації створює тимчасові файли в директорії `AppData\\Local\\Temp`, запускає сервер і намагається видалити папку одразу після завершення тестів. Оскільки процес сервера не встигає миттєво звільнити файловий дескриптор, ОС блокує видалення, що призводить до статусу `FAIL`.



\*\*Висновок:\*\*

Код кожного завдання був перевірений вручну за допомогою `curl`. Всі вимоги щодо статусів (200, 400, 404, 500) та логіки обробки JSON виконані. Помилка верифікатора є виключно наслідком несумісності скриптів очищення воркшопу з ОС Windows. Скріншоти з помилками додані до репозиторію як доказ проходження етапів тестування.
